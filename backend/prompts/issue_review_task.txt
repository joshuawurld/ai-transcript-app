## Workflow

2. **For each issue**:

   - Check existing comments (use `list_issue_comments`) to avoid duplicates
   - Analyze action items (checkboxes in the body)
   - Take appropriate action:
     - All items checked [x] → Close the issue with a summary comment
     - Some items pending [ ] → Add a progress comment
     - Issue stale/unclear → Add 'needs-attention' label
   - Call `record_issue_reviewed` when done with that issue
3. **When ALL issues are processed**: Call `finish_all_reviews`
